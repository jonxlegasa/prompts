# prompts

A collection of AI prompts ‚Äî system instructions, mission statements, and reusable skills ‚Äî with a stow-based deployment system for tools that need them.

## What this is

A place to collect and manage prompts: system prompts, interesting mission statements, reusable skills/commands, and anything else worth keeping. Some are actively deployed to AI tools, others are just saved for reference.

The deployment side uses GNU Stow to symlink prompts into the right locations for each tool. Currently supports Claude Code and OpenCode, but the system is tool-agnostic ‚Äî add a stow package for any tool.

| Source | Claude Code target | OpenCode target | Method |
|---|---|---|---|
| `src/MISSION.md` | `~/.claude/CLAUDE.md` | `~/.config/opencode/AGENTS.md` | compose |
| `src/skills/*.md` | `~/.claude/commands/*.md` | `~/.config/opencode/agents/*.md` | compose |
| `src/output-styles/*.md` | `~/.claude/output-styles/*.md` | `~/.config/opencode/output-styles/*.md` | symlink |
| `src/hooks/hooks.json` | `~/.claude/settings.json` | ‚Äî | symlink |
| `src/hooks/citation-audit.ts` | ‚Äî | `~/.config/opencode/plugins/citation-audit.ts` | symlink |

## Prerequisites

- [GNU Stow](https://www.gnu.org/software/stow/)

## Installation

```bash
git clone git@github.com:jonxlegasa/prompts.git
cd ~/prompts
bash setup.sh
```

`setup.sh` does two things:
1. `bash utils/compose.sh` ‚Äî copies `src/` prompts into the stow packages
2. `stow --adopt claude opencode` ‚Äî symlinks the packages into `~`

## Usage

Edit prompts in `src/`, then re-run setup:

```bash
vim src/MISSION.md        # edit the main system prompt
vim src/skills/my-cmd.md  # edit or add a skill
bash setup.sh             # rebuild + redeploy
```

## .gitignore

The generated files inside the stow packages are **not** tracked in git. They are build artifacts created by `compose.sh` from the source files in `src/`. The `.gitignore` excludes:

- `claude/.claude/CLAUDE.md` ‚Äî generated from `src/MISSION.md`
- `claude/.claude/commands/` ‚Äî generated from `src/skills/*.md`
- `opencode/.config/opencode/AGENTS.md` ‚Äî generated from `src/MISSION.md`
- `opencode/.config/opencode/agents/` ‚Äî generated from `src/skills/*.md`

The following are **not** gitignored ‚Äî they are tracked symlinks pointing back to `src/`:

- `claude/.claude/settings.json` ‚Üí `src/hooks/hooks.json`
- `claude/.claude/output-styles/` ‚Üí `src/output-styles/`
- `opencode/.config/opencode/output-styles/` ‚Üí `src/output-styles/`
- `opencode/.config/opencode/plugins/` ‚Üí `src/hooks/`

Only `src/` is the source of truth. On a fresh clone, run `bash setup.sh` to rebuild everything.
